About Our Project

For our Software Lab Project we decided to create a database focusing on board games, board game genres, and board game publishers.As individuals who spend a lot of time with friends playing video games, board games, and other group activities we determined that this database would be useful for friends to find games, genres, and publishers that they might enjoy based off of games they've played before.

As a team we initially delegated tasks so that each person was responsible for managing one model and worked to build up our vision for the site from there. In planning our project we discussed what information from our API resources would be relevant to our users and prioritized displaying that information. For Phase II, we shifted to dividing up work into API and database management, front-end coding, testing, which again changed as the project progressed and we resorted to more peer programming. We used html, css, JavaScript (jquery), and python (flask) for building this project and tested the site using selenium and python unittest. We used Bootstrap to implement many of the design features and the Google Cloud SDK to deploy the site.

For Phase I of our project we primarily used Board Game Atlas. Our other APIs for the project are Board Game Geek and Google Developer's Customer Search JSON API (for Board Game Companies). An interesting observation we made when searching for a reliable API was the majority of database sites for board game data cite Board Game Geek as the API they use. Some of the sites claiming to be their own API even waited until the acknowledgements to recognize that their data was sourced from the Board Game Geek API.
